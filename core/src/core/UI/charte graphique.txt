Ce document décrit la charte graphique qui m'a permis de définir l'interface graphique pour un noeud.

Lors de la construction d'un dataflow, un noeud peut être représenté de trois manières :
	- une version réduite qui défini une représentation minimale du noeud (title())
	- une version intermédiaire qui permet un contrôle ou un affichage restreint des propriétés du noeuds (thumbnail_representation)
	- une version étendue qui permet un contrôle total des paramêtres du noeud (edit_representation)

Deux fonctions supplémentaires fournissent des informations de décoration. La fonction color défini une couleur pour les interfaces utilisées par le noeud. L'utilisateur est vivement encouragé à ne pas redéfinir les couleurs des interfaces de bases telles que int,float,bool,.. La fonction string_representation fourni une description plus poussée et pourra être utilisée dans un tooltip par exemple.

A titre d'exemple, voila une représentation graphique possible des noeuds dans un dataflow :
	- version réduite :

|----------------|
| I I      | - x |
|----------------|
|     |  title   |
|----------------|
| O              |
|----------------|

	- version intermédiaire :

|----------------|
| I I      | - x |
|----------------|
|     |          |
|     |   user   |
|     |  defined |
|     |          |
|----------------|
| O              |
|----------------|

	- la version étendue est entièrement contrôlée par le créateur du noeud.
